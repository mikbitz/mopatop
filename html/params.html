Parameter Setting and Experiments

Parameter File

The model reads in its parameters from a file specifying one parameter value per line in the form

parameterName=value

where lines without and "=" or starting with a # are treated as comments. parameterNames are grouped together in terms
of common functions using a dot-separated form e.g. everything beginning "run." has to do with control of the model run
including things like run.nSteps for the number of model timesteps to be used or run.randomSeed for the initial random number seed e.g..

run.nSteps=5000

Where a parameter exists in the model but its value is not specified in the parameter file used, then a hard-coded default value
is used instead (see parameters.h). Any parameters set in the parameter file that do not have default values, or do not exist
will cause the model to halt. The file to be used is given when running the model as the first command-line argument as

modelexe parameterFileName \n

Every model run writes out a file listing the parameter values used in alphabetical order to the model output directory.
In this way many experiments with different parameter settings can be run without needing to edit or re-compile the model,
and a record of the model settings is kept with the results.

Experiment Files

In order to systematically explore and archive model behaviour, a facility for setting up experiments and keeping a list of them in a database
is included. The experiment database takes to form of a simple csv file listing the name of each experiment, the model version used, the experiment specification file
and a brief description - this is currently kept in a sub-directory name "experiments"
An example experiment is used to help with setup of new experiments including an example specification and base parameter file.
Two python programs, makeExperiment.py and runExperiment.py allow for new experiments to be created and run in a standardised way.
The experiment specifcation file is again a csv, this time including the experiment name, parameter file name and columns giving the
values of the parameters to be used in each model run, with the full parameter set being used for a run specified in each row of the
file. Parameters not mentioned in the specification file are assumed to take default values - any value that is varied must have a
value given for every run.
makeExperiment.py allows for a new experiment to be given a name, modelversion and description - the script then copies the example experiment
into a new directory, suitably re-named, and adds the experiment to the database. The parameter and specification files can then be edited, and the
entire run set executed using runExperiment.py (or else a full set of parameter files can be generated, in case, for exmaple, the experiment is to.
be run via a queueing system such as on HPC machines).
