Detailed Description

The model contains all the agents and places, and steps them through time.

At the moment time steps are not even in length (since the schedule values are not of the same length for work/home/transport)

Parameters
parametersA reference to a class that holds all the possible parameter settings for the model.
Using a reference ensures the values don't need to be copied

The relative structure of the places, size homes and workplaces and the number and size of transport vehicles, together with the schedule,
will jointly determine how effective the disease is a spreading, given the contamination rate and recovery timescale
This simple intializer puts three agents in each home, 10 agents in each workplace and 30 in each bus - so agents will mix in workplaces, home and buses in slightly different patterns.

void model::setOutputFilePaths ( parameterSettings &  parameters)
inline

Create the system of directories for model experiments and their outputs.

A system of directories gets created, starting from the top level experiment.output.directory.
To this is added the name of an experiment, which is assumed to consist of a number of separate runs with
different parameter values (e.g. random seed). By default a max. of 10000 runs is assumed - this gives tidy names
like ./output/experiment.test/run_0000 ./output/experiment.test/run_0001 ... ./output/experiment.test/run_9999
More then 10000 will be ok, but the directory names will spill over to ./output/experiment.test/run_10000 etc.
The number can be customised using experiment.run.prefix, which should be set to a power of 10.
A single run can be specified by setting experiment.run.number - if this coincides with and existing run/directory
then the files there may be overwritten (unless explicit output file names are changed).

Advance the model time step
.

split up the timestep into update of places, contamination of places by agents, infection and progress of disease and finally update of agent locations
These loops are separated so they can be individually timed and so that they can in principle be individually parallelised with openMP
Also to avoid any systematic biases, agents need to all finish their contamination step before any can get infected.

Parameters
stepNumberThe timestep number passed in from the model class

◆ randoms

std::vector<randomizer> model::randoms
private

multi threaded runs are faster if each thread has its own RNG

Each RNG is created with its own seed, seeds being separated by 1 incrementally. This means that multi-threaded runs
can be reproduced if the same number of threads is used (and the same start random seed), but runs with different numbers of threads
will typically produce different output (to the extent that output is stochastic)
