Detailed Description

The main agent class - each agent represents one person.

Agents move from place to place, using the travelSchedule. If they have the disease, the cough at each place they visit and contaminate it
If they are in a contaminated location, they may contract the disease. Additionally they may do other things in their current location.

Member Enumeration Documentation

◆ placeTypes

enum agent::placeTypes

This enum associates a set of integers with names, in order to identify types of place.

So home=0, work=1 etc. This allows meaningful names to be used to refer to the type of place the agent currently occupies, for example. Each agent has its own mapping from the placeType to an actual place - so home for agent 0 can be a different place for home for agent 124567. transport vehicles are places, albeit moveable!

◆ outwardTravel()

void agent::outwardTravel ( )

set up the schedule for travelling away

this sets a schedule and a set of places to visit. When there are multiple MPI domains, the copy of a remote agent needs to run this
with the local places so as the make sure the traveller agent gets a schedule with places that exist on this domain (see fetchall.h). The local agents also calls
this routine, which allows it to set local places (such as return flight) that will be ready for it on return, including if it comes back from a remote domain.

◆ placeCache

place* agent::placeCache[3]

a stack to store temporarily any places that need to be remebered for later use. Used when agent travels outside standart routine.

Visiting places using a remoteTravel.h object resets the places stored in places to point e.g. home and vehicle to holiday destinations
the cache allows original places to be remebered and restored on return from travel. Note that using an STL stack would work, but is hugely memory expensive.

An array of pointers to places.

  • indexed using the placeType, so that the integer value doesn't need to be used - instead one can use the name (home.work etc.)
