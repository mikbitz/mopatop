Introduction

This model is aimed at representing the patterns of movement and interaction of agents that represent individual people as they go about their daily activities.
The current version is intended to show how this can be done using a simple C++ program (using this language for speed of execution)
For this reason at the moment all code is in a single file. This can become unwieldy in a larger application though.
The current objective is to be able to model a simple disease, and to tie this to agent behaviour at the scale of an entire country.
Loop parallelisation with openMP is used to accelerate execution if needed.

Main ideas

Agents move between places according to a given fixed travel schedule. Places include transport vehicles.
In each place, agents with a disease can add contamination, which then decays exponentially over time
(or else it can be reset to zero at the start of each step - disease transmission then just depends on
number of agents in any location). The release of total contamination depends linearly on time spent in
each location. In a contaminated location, susceptible agents can pick up the infection - they then recover
with a fixed chance per timestep, and are subsequently immune.

Compiling the model

On a linux system with g++ installed just do
g++ -o agentModel -O3 -fopenmp -std=c++17 main.cpp
Note the current version requires g++>= and c++17 in order for the filesystem function to work (for creating new directories etc.)
If using openmp (parallelised loops) then set the number of threads in the parameterSettings class.
Note the number of cores to be used must be <= number supported by the local machine!

Running the model

At present this is a simple command-line application - just type the executable name (agentModel above) and then return.
This will use the default parameter file. To use an alternative specify the file name after the executable name:-

agentModel parameterFileName
