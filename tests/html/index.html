Introduction

This directory contains the unit tests for the mopatop model

Main ideas

The system uses the cppunit test system, version 1.14 to test each part of the model against some standard cases
For each class in the model, a header file is defined here containing tests of that class. Each header containes
a test suite, each of which runs through some of the methods defined in the classes to check that behave as expected
in a well defined and limited number of cases where the output can be known ahead of time. This helps to prevent some
of the more easily caught programming or conceptual errors. Each test suite is added to a testRunner defined in main
Note that because some of the classes can depend on each other, running some of the tests might potentially affect others
(e.g. the timeStep class is static, so if its parameters are altered, subsequent tests that depend on the timestep might
be altered). This should be borne in mind when writing new tests.
When modifications are made to any of the classes, tests should be re-run to chekc that nothing has been broken. Any new
code not subject to testing should have tests added here as necessary. The tests themsevles have their own documentation
generated by Doxygen and kept in the html sub-directory.

Compiling

On a linux system with g++ installed just do
g++ -otests -std=c++17 -lcppunit ../timestep.cpp ../places.cpp ../disease.cpp testing.cpp
Note the current version requires g++>= and c++17 in order for the filesystem function to work (for creating new directories etc.)

Running

At present this is a simple command-line application - just type the executable name (tests above) and then return.
This will use the parameter file called testParameterFile in this directory.

./tests

The parameter file allows for a number of tests that depend on parameter settings.
